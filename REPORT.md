# Отчёт: подготовка данных для обучения Whisper (башкирский язык)

Дата: 2026-07-19.

**Важно:** само обучение (`whisper.py`) на момент отчёта **не запускалось** —
ни разу не скачивался базовый чекпоинт `openai/whisper-medium`, в кэше
Hugging Face (`~/.cache/huggingface/hub`) есть только модель форс-алайнера
(`MahmoudAshraf/mms-300m-1130-forced-aligner`). Ниже — то, что сделано на
пути к обучению, фактическое состояние данных на диске, и что нужно
доделать, прежде чем запускать `whisper.py`.

## 1. Источники данных

Корпус строится из трёх независимых источников речи:

1. **Аудиокниги и рассказы** из Telegram-канала `bashaudiokitap` —
   собираются `scrape_tg_books.py` → `books/` (583 mp3) + `books_metadata.csv`.
2. **Стихи в исполнении школьников** (записи с YouTube, ссылки — в
   `bashkir_poems_bashkir.csv`) — скачиваются `download_bashkir_audio.py`.
3. **Диалектологические полевые записи** в формате ELAN (`elang_data/`) —
   готовые аудио+разметка, не требуют отдельного скачивания.

Плюс должен быть официальный **Mozilla Common Voice (Bashkir)** — но на
момент отчёта он **не скачан** (см. раздел 5).

## 2. Шаг 1 — сбор аудиокниг (`scrape_tg_books.py` → `books/`)

Выполнено: в `books/` лежит 583 mp3-файла. Статистика по ним (`books/stats.csv`,
собрана на основе `books_metadata.csv`):

- 214 уникальных авторов (часть — с пометкой «Билгеһеҙ автор», если пост в
  канале не содержал явного автора);
- 572 уникальные пары автор/название;
- покрытие метаданными: автор указан у 515 из 712 записей `books_metadata.csv`,
  название — у 687 из 712 (поле `reader`, диктор, не заполнено ни разу —
  распознавание диктора из тегов/подписи не сработало).

Это сырой материал для аудиокниг; сам по себе он **не является** обучающим
корпусом — нужен текст для сопоставления (шаг 3) и форс-алайнмент (шаг 4).

## 3. Шаг 2 — сбор текстов для сопоставления

Для части аудиокниг разыскивались тексты произведений:

- `find_author_works.py` — поиск по авторам из `authors.csv` на
  `kitaptar.bashkort.org` → `author_works.csv` (161 найденная ссылка на книгу).
- `download_pdfs.py` — скачивание самих PDF по этим ссылкам → `pdf/`.
- `download_kitaptar_collections.py` — отдельно докачаны собрания сочинений
  («Әҫәрҙәр», «Һайланма») → `kitaptar_collections/`.
- `search_kitaptar.py` — сверка уже собранных аудиокниг (`books_metadata.csv`)
  с библиотекой, чтобы понять, для каких вообще существует текст-первоисточник
  (`kitaptar_search_results.csv`).

Результат — папки `pdf/` и `kitaptar_collections/` с PDF/DjVu, из которых
вручную выбирались тексты, реально совпадающие с конкретными аудиофайлами
(зафиксировано в `pdf_audio_matches.md`).

## 4. Шаг 3 — сопоставление аудио и текста, укладка в `pairs/` и `pairs_poems/`

Из всего массива в 583 аудиокниг и найденных PDF **вручную** отобрана и
разложена по структуре `<Автор>/<Произведение>/{audio,pdf}` только часть:

- **`pairs/`** (проза): 8 авторов, 13 произведений, 48 исходных mp3.
- **`pairs_poems/`** (стихи, включая докачанные `download_bashkir_audio.py`
  ролики с YouTube): 46 авторов, 99 стихотворений, 142 исходных mp3.

Это значит, что в форс-алайнмент и, соответственно, в обучающий корпус
попало пока **48 из 583** аудиокниг из `books/` — основная часть `books/`
(без пары «текст найден и вручную сверен») в корпус ещё не включена. Это
главный резерв для расширения корпуса «книг».

Логи ручной чистки дублей авторов/названий папок (объединение вариантов
написания вроде «Мустай Карим» → «Мостай Кәрим») — `merge_log.txt`,
`merge_log2.txt`, `merge_titles_log.txt`, `merge_state.txt`, `inspect.txt`.

## 5. Шаг 4 — форс-алайнмент (`ctc-forced-aligner/`)

Для каждой пары `audio + текст` в `pairs/` и `pairs_poems/` прогонялся
сторонний инструмент `ctc-forced-aligner` (git-клон в корне проекта),
дающий word-level тайминги → `*.alignment.json` рядом с mp3. Именно этот
JSON потом режется на предложения в `pairs_to_common_voice.py` (шаг 6).
Использованная модель форс-алайнера уже в кэше HF
(`MahmoudAshraf/mms-300m-1130-forced-aligner`).

## 6. Шаг 5 — конвертация в Common Voice-формат

Три отдельных прогона конвертации, каждый — `clips/*.wav` + `train.tsv`:

```
python pairs_to_common_voice.py --input-dir pairs --output-dir common_voice_pairs
python pairs_to_common_voice.py --input-dir pairs_poems --output-dir common_voice_pairs_poems --speaker-mode filename
python eaf_to_common_voice.py --input-dir elang_data --output-dir common_voice_elang
```

(логи прогонов — `convert_pairs.log`, `convert_pairs_poems.log`; в первом
встречаются ошибки декодирования отдельных mp3 через `mpg123`/`vcpkg`
— конвертер их пропускает, не роняя весь прогон).

## 7. Официальный Mozilla Common Voice — не готов

`whisper.py` по умолчанию ожидает также каталог `common_voice` (единственный
источник с настоящими `train/dev/test`-сплитами, а не «вырезанным» eval).
На диске такого каталога нет — есть только осиротевший `train.tsv` в корне
проекта (119 133 строки в исходном формате Common Voice: `client_id, path,
sentence, ...`), без папки `clips/`. Файл, судя по всему, — остаток
незавершённого скачивания. Прежде чем обучать модель с включением
официального датасета, нужно повторно скачать его:

```
python download_common_voice_mdc.py            # через Mozilla Data Collective (нужен MDC_API_KEY)
# либо
python download_dataset.py                     # через Hugging Face datasets (нужен HF_TOKEN)
```

## 9. Блокеры перед запуском обучения

1. **Зависимости.** Доустановить недостающее:
   `pip install torchaudio accelerate jiwer evaluate matplotlib`.
2. **ffmpeg.** Не найден в `PATH` — нужен для декодирования mp3-клипов при
   обучении. Поставить и добавить в `PATH`, либо (как уже сделано в
   `download_bashkir_audio.py`) использовать портативный бинарник из
   `imageio-ffmpeg`, адаптировав под это `whisper.py`.
3. **Устройство обучения.** Установленный `torch` собран под Intel **XPU**,
   CUDA недоступна (`cuda.is_available() == False`). `whisper.py` жёстко
   прописывает CUDA-специфичные опции (`bf16=True`, `tf32=True`,
   `optim="adamw_torch_fused"` в `Seq2SeqTrainingArguments`) — их нужно
   будет адаптировать под XPU (или явно посчитать, что реалистично обучать
   `whisper-medium` на CPU — на практике непрактично).
4. **Официальный Common Voice** отсутствует (раздел 7) — либо скачать его,
   либо осознанно обучать только на трёх «доморощенных» корпусах
   (`common_voice_pairs`, `common_voice_pairs_poems`, `common_voice_elang`,
   в сумме ~60.5 часов).
5. **Неполное покрытие `books/`.** Только 48 из 583 аудиокниг прошли
   сопоставление с текстом и форс-алайнмент; остальные — потенциальный
   резерв для расширения `pairs/`, но требуют ручной работы (найти текст,
   сверить, прогнать `ctc-forced-aligner`).

## 10. Следующий шаг (команда для запуска, ещё не выполнялась)

После снятия блокеров из раздела 9 обучение запускается так:

```
python whisper.py \
  --data-dirs common_voice_pairs common_voice_pairs_poems common_voice_elang \
  --model openai/whisper-medium \
  --output-dir whisper-ba
```

(`common_voice` намеренно не включён в `--data-dirs`, пока не скачан —
см. раздел 7). Скрипт сам скачает базовый чекпоинт `openai/whisper-medium`
с HF Hub при первом запуске, объединит train/eval сплиты всех трёх
каталогов (eval для каждого будет вырезан как 2% от `train.tsv`, поскольку
ни один из трёх не содержит `dev.tsv`), закэширует извлечение признаков
по чанкам в `whisper-ba/feature_cache/`, и по завершении обучения сам
прогонит `evaluate_test_sets.py` — но только по каталогам с настоящим
`test.tsv`, которого пока ни у одного из трёх нет, так что финальная
метрика на held-out тесте на этом наборе данных получена не будет.
